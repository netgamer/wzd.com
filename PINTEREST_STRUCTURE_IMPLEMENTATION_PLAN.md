# WZD Pinterest형 구조 구현 1차 계획

## 이번 구현 라운드 목표
문서 기준을 실제 앱에 적용하는 첫 라운드로, 가장 위험한 회귀를 먼저 복구하면서 Pinterest형 구조의 핵심 뼈대를 도입한다.

## 이번 라운드 범위
1. 모바일에서 보드 콘텐츠가 안 보이는 회귀 버그 복구
2. 모바일 상단을 보드 선택 중심으로 단순화
3. 모바일 하단 메뉴의 기본 구조 도입
4. PC 상단에서 검색/과한 헤더 노출을 약화하고 보드 선택 중심 구조로 정리할 준비

## 이번 라운드 금지 범위
- 모든 카드/위젯 완전 리디자인
- 테마 6종 전면 적용
- 데이터 로딩 로직 변경
- 데스크톱 전체 레이아웃 전면 개편

## 회귀 체크리스트
- 모바일에서 보드 탭이 보이는가
- 모바일에서 보드 내용이 실제로 보이는가
- 모바일에서 제목/핵심 컨트롤이 사라지지 않는가
- 모바일에서 하단 메뉴가 콘텐츠를 가리지 않는가
- PC에서 기존 보드 화면이 완전히 깨지지 않는가
