# WZD 공모전용 업그레이드 검증 보고서

## 1. 지금 해결해야 하는 실제 문제
4월 15일 마감인 모두의 아이디어 공모에 WZD를 제출하려면, 현재 서비스와 설명이 심사위원에게 더 빠르고 선명하게 읽혀야 한다. 지금 WZD는 방향이 많이 정리되었지만 여전히 다음 리스크가 남아 있다.

- 기능이 많아 보여 제품 정의가 퍼질 수 있음
- 심사위원이 5초 안에 `이게 정확히 뭔가`를 이해하기 어려울 수 있음
- 실제 서비스 표면과 공모전 설명 문구가 아직 완전히 일치하지 않을 수 있음

즉 지금 필요한 것은 기능을 더 늘리는 것이 아니라, **공모전 관점에서 이해도와 설득력을 높이는 업그레이드**다.

## 2. 왜 지금 중요한가
### 매우 중요함
- 마감이 4월 15일로 가까움
- 지금까지의 개발 자산을 제출 가치가 있는 형태로 정리해야 함
- 기능 추가보다 메시지 정렬과 대표 화면 정리가 더 높은 효율을 냄

### 지금 하지 않으면 생길 문제
- 좋은 제품이어도 심사위원에게 퍼져 보일 수 있음
- 기능은 많은데 핵심이 약한 인상으로 남을 수 있음
- 발표자료와 실제 화면이 따로 노는 느낌이 날 수 있음

## 3. 누가 가장 먼저 이 업그레이드의 도움을 받는가
1. 공모전 심사위원
2. 제출 자료를 보는 외부 평가자
3. WZD를 처음 보는 잠재 사용자

즉 이번 업그레이드는 일반 사용자보다 먼저 **심사자 이해도 최적화**를 목표로 해야 한다.

## 4. 리스크
### 리스크 1: 공모전용으로 너무 과장하면 실제 제품과 어긋날 수 있음
그래서 비전은 말하되, 현재 구현과 자연스럽게 이어져야 한다.

### 리스크 2: 많은 기능을 계속 보여주면 오히려 약해짐
공모전에서는 기능 다양성보다 문제-해결-차별점이 더 중요하다.

### 리스크 3: 디자인보다 메시지가 더 중요하지만, 화면이 산만하면 메시지가 안 들어옴
따라서 공모전용 대표 화면 정리는 필요하다.

## 5. 추천 MVP 범위
공모전 전까지 꼭 해야 할 최소 범위는 아래다.

### A. 제품 정의 고정
- WZD를 `브라우저를 열면 가장 먼저 보고 싶은 개인화 첫 페이지`로 고정
- 북마크 + RSS를 현재 코어로 강조
- 링크페이지/윈도우 위젯/모바일 동기화는 확장성으로만 제시

### B. 랜딩/대표 화면 정리
- 첫 화면에서 개인화 첫 페이지라는 점이 더 즉시 읽히게 수정
- 대표 예시는 북마크 + RSS 중심으로 압축
- 산만한 샘플은 줄이거나 뒤로 미룸

### C. 공모전 제출용 정렬
- 제출 문서 / PPT / 실제 제품 메시지를 동일한 문장 구조로 통일

## 6. 지금 미뤄야 할 것
- 새로운 기능 추가
- 위젯 확장 경쟁
- 마켓/에이전트 전면화
- 하드웨어/앱 비전 구현
- 공모전 이후에도 되는 큰 구조 개편

지금은 제품을 더 크게 만드는 단계가 아니라, **더 선명하게 보이게 만드는 단계**다.

## 7. 성공 검증 기준
이번 업그레이드가 성공했다고 보려면 아래가 충족되어야 한다.

1. 심사위원이 첫 화면과 제목만 보고 WZD를 `개인화 첫 페이지`로 이해할 수 있다.
2. 북마크와 RSS가 현재 핵심이라는 점이 보인다.
3. 메모보드/생산성 대시보드/AI 허브처럼 퍼져 보이는 인상이 줄어든다.
4. 제출 문서와 실제 화면이 같은 제품을 말하고 있다는 느낌이 든다.

## 최종 판단
### PASS
지금 WZD를 공모전용으로 업그레이드하는 것은 매우 중요하고, 지금 바로 해야 한다.

단, 방향은 명확하다.
- 더 많은 기능이 아니라
- 더 선명한 메시지
- 더 대표적인 화면
- 더 일관된 제출 자료

이 세 가지에 집중해야 한다.
