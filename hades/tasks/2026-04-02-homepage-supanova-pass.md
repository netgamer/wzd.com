## Goal

Supanova 기준으로 홈 랜딩을 다시 정리해 카드 나열 느낌을 줄이고, 프리미엄한 히어로/섹션 리듬이 한눈에 드러나게 만든다.

## User-visible outcome

- 홈 첫 화면에서 서비스 성격이 더 선명하게 읽히고, 히어로가 일반 카드가 아니라 메인 랜딩 섹션처럼 보인다.
- 기능, 설명, 프로세스, CTA 구간이 서로 다른 레이아웃과 무드로 보여 카드 복붙 같은 인상이 줄어든다.
- 모바일에서도 한 장짜리 랜딩처럼 자연스럽게 이어지고 버튼/카드 가독성이 유지된다.

## In scope

- `src/styles/main.css`의 홈 랜딩 전용 레이아웃, 배경, 카드 계층, 반응형 규칙
- `src/App.tsx`의 홈 랜딩 문구 중 필요 최소한의 구조/카피 보정
- 홈 화면 데스크톱/모바일 시각 검증

## Out of scope

- 작업공간 `/#workspace` UI
- 인증, 저장, 보드 데이터 모델, 링크 프리뷰 동작
- 홈 랜딩 외 다른 템플릿 보드의 문구 일괄 수정

## Risks

- 홈 랜딩 전용 CSS가 공유/보드 페이지 규칙과 충돌할 수 있음
- 섹션 재배치가 모바일에서 과한 높이로 이어질 수 있음
- 문구를 다듬을 경우 기존 의도와 톤이 일부 바뀔 수 있음

## Pass criteria

- `1280px+`: 히어로가 일반 카드처럼 보이지 않고, 기능/설명/CTA 섹션이 서로 다른 구도와 밀도로 구분된다.
- `768px-1279px`: 홈 카드 배치가 2열 또는 혼합 폭으로 유지되며, 특정 행이 전부 동일 폭 카드로만 반복되지 않는다.
- `767px 이하`: 모든 홈 섹션이 1열로 자연스럽게 이어지고, 제목/본문/버튼이 겹치거나 잘리지 않는다.
- 홈 상단과 본문 패널이 동일한 베이지 카드 덩어리처럼 붙어 보이지 않는다.
- `npm run build` passes
- `/gstack-review` finds no blocking issues
