# Task Template

## Goal

README의 제품 설명을 WZD의 더 뾰족한 방향으로 다시 쓴다.

## User-visible outcome

- README 첫머리에서 WZD가 어떤 문제를 푸는지 더 명확하게 이해할 수 있다.
- LinkTree 대체 개인화 페이지와 관심사별 RSS/콘텐츠 보드라는 두 핵심 축이 문서에 반영된다.

## In scope

- `README.md`의 제품 소개 문구

## Out of scope

- 코드 변경
- 배포/설정 절차 변경

## Risks

- 제품 포지셔닝 문장이 너무 길어질 수 있다.
- 기능 나열식 문서가 되어 핵심 메시지가 흐려질 수 있다.

## Pass criteria

- README 상단에 WZD의 제품 정체성이 한두 문단으로 분명하게 드러난다.
- LinkTree 대체 개인화 페이지와 RSS/콘텐츠 보드 사용 사례가 구체적으로 설명된다.
- 기존 실행/배포 가이드는 유지된다.

## Required checks

- `npm run build`
- `/gstack-review`

## Extra checks if applicable

- `/gstack-qa-only`
