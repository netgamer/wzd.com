## Goal

새 보드 버튼을 즉시 생성이 아닌 선택형 생성 모달로 바꾸고 무료/유료 보드 개수 제한을 적용한다.

## User-visible outcome

- `새 보드`를 누르면 바로 생성되지 않고 생성 모달이 열린다.
- 사용자는 빈 보드 또는 테마가 적용된 빈 보드를 고른 뒤에만 보드를 만든다.
- 무료 플랜은 보드 최대 3개, 유료 플랜은 최대 10개 제한이 모달에서 보인다.
- 모달 안에서 `$5`, `$10`, `비즈니스 문의` 옵션을 확인하고 플랜을 바꿀 수 있다.

## In scope

- `src/App.tsx`의 보드 생성 흐름, 플랜 상태, 생성 제한 로직
- `src/styles/main.css`의 보드 생성 모달과 플랜 카드 스타일
- `hades/tasks/2026-04-15-rework-board-creation-and-plan-limits.md`

## Out of scope

- 실제 결제 연동
- 서버 영구 구독 상태 저장
- 비즈니스 문의 백엔드 처리

## Risks

- 로컬 플랜 상태와 실제 결제 상태는 아직 분리되어 있다.
- 기존 템플릿 진입 구조를 너무 크게 바꾸면 다른 생성 경로가 깨질 수 있다.

## Pass criteria

- `새 보드` 클릭 시 생성 모달이 열린다.
- 모달에서 `빈 보드` 또는 `테마 보드`를 고른 뒤에만 보드가 생성된다.
- 무료 플랜에서 활성 보드가 3개면 추가 생성이 막히고 이유가 모달에 보인다.
- `$5` 또는 `$10` 플랜을 선택하면 최대 보드 수가 10개로 표시되고 생성 가능 상태가 바뀐다.
- `보드 복제`로도 생성 제한을 우회할 수 없다.
- `981px+`: 생성 모달 카드가 2열 이상으로 정리된다.
- `980px and below`: 생성 모달이 한 열로 줄고 버튼이 잘리지 않는다.
- `npm run build` passes
