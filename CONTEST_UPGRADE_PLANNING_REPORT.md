# WZD 공모전용 업그레이드 기획 보고서

## 1. 이번 라운드 목표
4월 15일 공모전 제출 전까지 WZD를 심사위원 관점에서 더 선명하게 보이게 만든다.

### 한 문장 목표
**WZD를 `브라우저를 열면 가장 먼저 보고 싶은 개인화 첫 페이지`로 더 명확하게 읽히도록 랜딩과 대표 화면을 정리한다.**

## 2. 이번 라운드 핵심 전략
이번 라운드는 새 기능을 추가하는 라운드가 아니다. 이미 있는 기능과 화면 중에서 공모전 제출에 유리한 것만 앞으로 끌어오고, 퍼져 보이는 요소는 약화시키는 라운드다.

### 핵심 전략
1. 메인 메시지 강화
2. 대표 샘플 구성 정리
3. 제출 문구와 실제 화면의 일치 강화

## 3. In scope
### 1) 랜딩 대표 샘플 정리
- 북마크 + RSS + 개인화 첫 페이지 느낌을 강하게 주는 카드만 전면에 남김
- 생산성/운영/팀 협업/AI 허브처럼 퍼져 보이는 샘플은 축소 또는 후순위 배치

### 2) Hero/preview 보강
- 첫 화면에서 `브라우저 첫 페이지`라는 인상이 더 강하게 오도록 샘플 구성 조정
- `내가 매일 여는 페이지`라는 감각을 더 살림

### 3) 공모전용 연결성 강화
- 제출 문서와 실제 화면에서 같은 제품을 말하고 있다는 느낌을 만들기
- 필요하면 소규모 카피 정리 포함

## 4. Out of scope
- 새 기능 추가
- 위젯 신규 개발
- 마켓/에이전트 기능 개편
- 윈도우/모바일 위젯 구현
- Supabase 구조 변경
- 게시판/업데이트 허브 확장 작업

## 5. 우선 변경 대상
- `src/features/landing/LandingPage.tsx`
- 필요 시 `src/styles/main.css`

## 6. 이번 라운드에서 개발자가 해야 할 일
### 개발자 태스크
- 랜딩 샘플 카드 구성에서 북마크/RSS 중심성이 더 드러나게 정리
- 너무 많은 위젯/운영형 샘플을 줄이거나 뒤로 배치
- 첫인상이 `작업 보드`보다 `개인화 첫 페이지`에 가깝게 보이도록 조정
- build 통과 후 commit/push

## 7. 완료 기준
1. 랜딩 샘플을 볼 때 북마크/RSS가 현재 코어라는 점이 더 잘 느껴진다.
2. 생산성 보드/협업 보드/운영 대시보드처럼 퍼져 보이는 인상이 줄어든다.
3. 공모전 문서의 정의와 실제 랜딩 화면이 더 자연스럽게 맞물린다.
4. build가 통과한다.

## 최종 정리
이번 라운드는 기술 확장이 아니라 **심사자 이해도 최적화 라운드**다.

즉 해야 하는 일은 하나다.

**WZD가 지금 무엇인지 더 빨리, 더 선명하게 보이게 만드는 것.**
