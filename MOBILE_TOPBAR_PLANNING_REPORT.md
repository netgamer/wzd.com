# WZD 모바일 상단영역 기획 보고서

## 1. 이번 라운드 목표
모바일에서 상단영역이 더 단순하고 안정적으로 보이도록 정리한다.

### 한 문장 목표
**모바일 상단영역의 정보 밀도를 줄이고, 저장 상태와 주요 액션의 시각 우선순위를 더 명확하게 만든다.**

## 2. In scope
- 모바일 상단영역의 레이아웃/간격/문구 정리
- 저장 상태 메시지 표현 방식 완화
- 탭/버튼/정보 블록의 시각 우선순위 조정
- 필요한 경우 관련 CSS만 최소 수정

## 3. Out of scope
- 데스크톱 헤더 전체 재설계
- 보드 기능 변경
- 저장 로직 변경
- 탭 구조 전체 변경
- 새 기능 추가

## 4. 우선 변경 대상
- `src/App.tsx`
- `src/styles/main.css`

## 5. 개발자 태스크
- 모바일 화면에서 상단영역이 차지하는 시각적 부담을 줄인다.
- 저장 상태 메시지를 덜 공격적으로 보이게 조정한다.
- 주 액션과 제목, 탭이 더 쉽게 읽히도록 간격과 구조를 다듬는다.
- build 통과 후 commit/push.

## 6. 완료 기준
1. 모바일 스크린샷 기준 상단이 덜 복잡해 보인다.
2. 저장 상태 노출이 더 차분하고 신뢰감 있게 느껴진다.
3. 사용자 시선이 제목 → 핵심 상태 → 주요 액션 순으로 자연스럽게 흐른다.
4. build가 통과한다.

## 최종 정리
이번 라운드는 기능 개발이 아니라 **모바일 첫인상 정리 라운드**다.
