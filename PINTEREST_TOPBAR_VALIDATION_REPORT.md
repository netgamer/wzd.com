# WZD Pinterest형 상단바 1차 검증 보고서

## 1. 실제 문제
현재 상단영역은 여전히 보드 선택 외 정보와 컨트롤이 섞여 있어 Pinterest형 콘텐츠 우선 구조와 완전히 맞지 않는다.

문제 요약:
- 상단이 보드 선택 전용 바처럼 읽히지 않는다.
- 제목, 상태, 액션, 검색 또는 기타 컨트롤이 혼재될 여지가 남아 있다.
- 사용자가 콘텐츠보다 상단 구조를 먼저 해석해야 한다.

## 2. 왜 지금 중요한가
- Pinterest형 전환에서 상단 정리는 가장 핵심적인 첫 단계다.
- 상단이 단순해져야 콘텐츠가 전면으로 나온다.
- 이후 좌측 메뉴/하단 메뉴/카드 통일 작업도 쉬워진다.

## 3. 이번 라운드 목표
- 상단을 `보드 선택 전용 바`에 가깝게 정리
- 보드 탭 중심 구조를 강화
- 상단의 보조 정보와 액션을 약화하거나 아래/다른 영역으로 밀어냄

## 4. 리스크
- 상단을 너무 줄이면 제목/핵심 구조 인지가 다시 약해질 수 있다.
- 보드 선택 UX가 오히려 나빠지면 안 된다.
- 데스크톱/모바일 차이를 한 번에 과하게 건드리면 회귀가 생길 수 있다.

## 5. 추천 MVP 범위
- App topbar 구조 최소 수정
- CSS로 상단 높이/시각 우선순위 정리
- 보드 탭이 가장 먼저 읽히도록 조정

## 6. 회귀 체크리스트
- 모바일에서 보드 탭이 보이는가
- 모바일에서 보드 본문이 보이는가
- 데스크톱에서 보드 선택 구조가 깨지지 않는가
- 제목/핵심 구조 인지가 완전히 사라지지 않는가
- build 통과
