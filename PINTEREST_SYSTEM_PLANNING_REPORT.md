# WZD Pinterest형 디자인 시스템 기획 보고서

## 1. 이번 라운드 목표
WZD를 Pinterest형 콘텐츠 우선 제품으로 정리하기 위한 디자인 시스템과 테마 시스템의 기준 문서를 먼저 만든다.

### 한 문장 목표
**카드, 색, 타이포, 간격, 네비게이션, 테마를 하나의 WZD 디자인 시스템으로 먼저 고정한다.**

## 2. In scope
- `WZD_DESIGN_SYSTEM_V1.md` 작성
- `WZD_THEME_SYSTEM_V1.md` 작성
- Pinterest형 PC/모바일 정보구조 기준 정의
- 카드/위젯/네비게이션/컬러/간격 원칙 문서화

## 3. Out of scope
- 실제 UI 전체 구현
- 모든 화면 즉시 리팩터링
- 테마별 레이아웃 변경
- 애니메이션 고도화

## 4. 핵심 산출물
### A. WZD_DESIGN_SYSTEM_V1.md
- 디자인 철학
- 레이아웃 원칙
- 카드 시스템
- 컬러 토큰
- 타이포 시스템
- spacing/radius/shadow
- 버튼/액션 규칙
- PC/모바일 내비게이션 원칙

### B. WZD_THEME_SYSTEM_V1.md
- 6개 테마 정의
- 각 테마의 배경/표면/텍스트/강조색
- 사용 원칙
- 금지사항

## 5. 후속 구현 라운드 분리안
1. 모바일 보드 가시성 회귀 복구
2. 상단/좌측/하단 네비 구조 재편
3. 카드 스타일 통일
4. 위젯 스타일 Pinterest화
5. 6개 테마 적용

## 6. 완료 기준
1. 시스템 문서 2개가 작성된다.
2. Pinterest형 구조 원칙이 명확히 정의된다.
3. 위젯은 카드형 콘텐츠 표면으로 규정된다.
4. 6개 테마가 구조가 아닌 톤 차이로 정리된다.

## 최종 정리
이번 라운드는 구현보다 앞선 **설계 잠금 라운드**다. 문서가 먼저 고정되어야 이후 Codex 작업도 덜 흔들린다.
