# WZD 실행 로드맵

`PRODUCT_PLAN.md`를 실제 개발 우선순위로 옮긴 실행 문서.

## 현재 목표

WZD를 "노션보다 가볍고, 메모앱보다 강한 개인화 보드"로 완성한다.

핵심 경험은 아래 네 가지다.

1. 붙여넣기가 빠르다
2. 카드가 보기 좋다
3. 공유가 쉽다
4. 모바일에서 자연스럽다

---

### 완료

- 링크/이미지/텍스트 붙여넣기 안정화

- 스타터 보드 템플릿

## 실행 순서

## Step 1. 보드 히스토리

### 목표

실수해도 복구할 수 있다는 신뢰를 만든다.

### 구현 범위

- 보드 설정에 `히스토리` 섹션 추가
- 현재 보드 상태를 스냅샷으로 저장
- 최근 스냅샷 목록 표시
- 선택한 스냅샷으로 보드 상태 복구
- 저장 개수 제한 적용

### 상태

- 완료

---

## Step 2. 보드 복제

### 목표

좋은 보드 구조를 새 보드의 출발점으로 빠르게 재사용하게 한다.

### 구현 범위

- 현재 보드 복제 버튼
- 메모, 배경, 설정, 레이아웃 스타일 복사
- 제목 자동 보정
- 공유 슬러그/휴지통/히스토리는 새 보드에 딸려가지 않게 정리

### 완료 기준

- 복제된 보드가 즉시 선택되고 편집 가능
- 원본과 독립적으로 수정 가능

---

## Step 3. 공유 페이지 강화

### 목표

공유 링크가 단순한 읽기 페이지가 아니라 정돈된 개인 페이지처럼 느껴지게 한다.

### 구현 범위

- 보드 커버/소개 영역
- 공유용 강조 테마
- 대표 카드/커버 이미지 지정

### 완료 기준

- 공유 링크 첫 인상이 강해짐
- 카카오톡/메신저 미리보기가 더 좋아짐

---

## Step 4. 프리미엄 가치 만들기

### 목표

결제를 붙이기 전에 "왜 결제해야 하는지"를 분명하게 만든다.

### 후보 기능

- 히스토리 고급 버전
- 고급 공유 테마
- 보드 복제 무제한
- 프리미엄 템플릿
- 업로드 용량 확장

---

## 기능 판단 체크리스트

새 기능을 넣기 전에 항상 아래를 확인한다.

1. 입력이 더 빨라지는가
2. 결과가 더 보기 좋아지는가
3. 공유가 더 쉬워지는가
4. 모바일에서 더 자연스러운가
5. 제품이 더 무거워지지 않는가

---

## 검증 원칙

기능을 넣고 나면 최소한 아래를 확인한다.

- `npm run build` 성공
- 데스크톱/태블릿/모바일 주요 레이아웃 확인
- 공유/읽기 전용/소유자 모드 확인
- 로컬 상태와 클라우드 저장 흐름 확인
