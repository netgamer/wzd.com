# WZD 실행 로드맵

`PRODUCT_PLAN.md`를 실제 개발 순서로 옮긴 실행 문서다.

## 목표

WZD를 “노션보다 가볍고, 메모앱보다 강한 개인화 보드”로 느끼게 만드는 것.

핵심 경험은 아래 네 가지다.

1. 붙여 넣기가 빠르다
2. 자동으로 카드처럼 정리된다
3. 공유했을 때 보기 좋다
4. 모바일에서도 편하다

---

## 지금부터의 운영 원칙

### 1. 기능보다 경험을 먼저 다듬는다

새 기능을 많이 넣기보다,
- 붙여 넣기
- 카드 표시
- 공유
- 모바일 조작

이 네 가지를 먼저 매끈하게 만든다.

### 2. 무거운 문서 도구처럼 가지 않는다

우선순위를 낮추는 것:
- 블록 에디터 확장
- 데이터베이스 빌더
- 복잡한 권한 체계

우선순위를 높이는 것:
- 카드화
- 시각적 정리
- 공유 링크
- 초대 기반 협업

### 3. 수익화보다 재방문 이유를 만든다

결제는 나중이다.
먼저 “이 앱을 계속 켜두고 싶다”는 이유를 만들어야 한다.

---

## 4주 실행 계획

## Week 1. 수집 경험 완성

### 목표

사용자가 아무 생각 없이 붙여 넣어도 기대한 결과가 나오게 한다.

### 작업

- 링크 붙여 넣기 안정화
- 캡처 이미지 붙여 넣기 안정화
- 이미지 + URL 동시 보존 안정화
- 잘못된 URL 자동 정규화
- 실패 시 화면 전체가 죽지 않도록 방어
- 첫 화면/빈 상태 문구 정리

### 완료 기준

- 링크, 이미지, 텍스트를 각각 붙여 넣어도 안정적으로 카드 생성
- 이미지와 URL이 함께 저장됨
- 붙여 넣기 실패 후에도 앱이 깨지지 않음

---

## Week 2. 공유 경험 완성

### 목표

보드를 공유했을 때 “정리된 개인 페이지”처럼 느껴지게 만든다.

### 작업

- 공유 URL 진입 안정화
- 소유자/편집자는 공유 링크로 들어와도 편집 가능
- 비로그인 사용자는 읽기 전용 보기
- 공유용 메타데이터(제목/설명/대표 이미지) 강화
- 공유 화면의 버튼/액션 정리
- 보드 초대와 공동편집 흐름 안정화

### 완료 기준

- 공유 링크가 비어 보이지 않음
- 권한에 따라 읽기/편집이 정확히 나뉨
- 카카오톡/메신저 공유 시 미리보기 정보 표시

---

## Week 3. 모바일 경험 완성

### 목표

모바일에서 보드를 탐색하고 편집하는 흐름이 앱처럼 자연스럽게 느껴지게 한다.

### 작업

- 모바일 상단바 단순화
- 햄버거 메뉴 정리
- 좌우 스와이프 보드 전환 다듬기
- 스와이프 중 옆 보드 미리보기 품질 개선
- 보드 전환 시 카드 크기/레이아웃 점프 제거
- 모바일 편집 버튼 밀도 최적화

### 완료 기준

- 스와이프 중/완료 후 레이아웃이 자연스럽다
- 상단 버튼 과밀도가 줄어든다
- 한 손으로 주요 기능 접근 가능

---

## Week 4. 정리 경험 강화

### 목표

수집한 카드가 쌓여도 지저분해지지 않게 만든다.

### 작업

- 보드 자동 정리 설계
- 카드 정렬 기준 정의
- 비슷한 카드 묶기 설계
- 템플릿 보드 초안
- 보드 설정 구조 개선

### 완료 기준

- 사용자가 “정리하기” 기능의 가치를 바로 이해할 수 있음
- 템플릿 기반 시작 경험이 가능함

---

## 즉시 착수할 우선순위

### P0

- 붙여 넣기 안정성
- 공유 링크 진입 안정성
- 모바일 보드 전환 품질

### P1

- 보드 초대/공동편집
- 빈 상태/가이드 카피 개선
- 설정/휴지통 정보 구조화

### P2

- 자동 정리
- 템플릿
- 보드 커버/OG 이미지 고도화

---

## 기능 판단 체크리스트

새 기능을 추가하기 전에 아래 다섯 개를 본다.

1. 입력이 더 빨라지는가?
2. 결과물이 더 보기 좋아지는가?
3. 공유가 더 쉬워지는가?
4. 모바일에서 더 편해지는가?
5. 노션처럼 무거워지지 않는가?

세 개 이상 만족하지 못하면 우선순위를 낮춘다.

## 바로 다음 개발 후보

### 후보 A. 링크 카드 고도화

- 링크 붙여 넣기 시 썸네일/도메인/제목 보강
- 링크 카드의 읽기성과 공유 품질 향상

### 후보 B. 자동 정리 버튼

- 카드들을 적절한 간격으로 다시 배열
- 색상/타입 기준으로 느슨하게 그룹핑

### 후보 C. 템플릿 스타터 보드

- AI 자료 모음
- 유튜브 기획
- 공부 보드
- 여행/쇼핑 보드

## 성공 기준

사용자가 WZD를 열었을 때 이렇게 느끼면 성공이다.

- “붙여 넣기 진짜 편하다”
- “대충 넣었는데 예쁘다”
- “이건 공유하기 좋다”
- “폰에서도 쓸 만하다”

이 감각이 만들어진 뒤에 결제를 붙인다.
