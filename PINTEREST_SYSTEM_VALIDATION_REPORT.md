# WZD Pinterest형 디자인 시스템 검증 보고서

## 1. 실제 문제
현재 WZD는 화면과 위젯별 스타일 규칙이 충분히 통일되지 않아 전체 인상이 중구난방하게 보인다.

주요 문제:
- 카드 반경, 패딩, 높이, 그림자 규칙이 들쭉날쭉하다.
- 색 사용 기준이 약해 위젯마다 따로 노는 인상이 난다.
- 모바일/PC의 정보구조가 일관되지 않아 제품 언어가 분산된다.
- 상단, 보드 탭, 위젯, 액션 카드가 하나의 시스템보다 화면별 예외처럼 보인다.

즉 지금 필요한 것은 부분 수정이 아니라 **Pinterest형 콘텐츠 우선 디자인 시스템**의 정의와 적용이다.

## 2. 왜 지금 중요한가
- 공모전 제출 전 제품 인상을 정리해야 한다.
- 모바일 회귀를 반복 수정하는 것보다 시스템을 먼저 세우는 편이 더 안정적이다.
- WZD의 핵심이 `개인화된 첫 페이지`라면, 시선이 설명보다 콘텐츠에 먼저 가야 한다.

## 3. 이번 라운드 핵심 방향
- 기본 정보구조를 Pinterest형으로 재정리
- 보드 선택은 상단에 고정
- PC 메뉴는 왼쪽, 모바일 메뉴는 하단
- 위젯은 관리 패널 느낌이 아니라 콘텐츠 카드처럼 정리
- 6개 테마는 같은 구조 위의 톤 차이로 제공

## 4. 리스크
- 시스템 정의 없이 바로 구현하면 다시 회귀가 생길 수 있다.
- 테마를 구조 변화까지 포함시키면 복잡도가 급증한다.
- 카드 스타일 통일 작업이 너무 넓어지면 공모전 직전 일정이 흔들릴 수 있다.

## 5. 추천 MVP 범위
1. 디자인 시스템 문서 v1 작성
2. 테마 시스템 문서 v1 작성
3. Pinterest형 정보구조(PC/모바일) 문서화
4. 이후 구현 라운드는 매우 잘게 분리

## 6. 지금 미뤄야 할 것
- 모든 화면의 동시 재디자인
- 카드 타입 무한 확장
- 고급 애니메이션/미세 효과 집착
- 테마별 레이아웃 차별화

## 7. 성공 기준
1. WZD의 카드/색/간격/타이포 기준이 문서로 고정된다.
2. 6개 테마가 같은 구조 위의 컬러 시스템으로 정의된다.
3. PC/모바일 정보구조 원칙이 통일된다.
4. 이후 구현 작업을 더 잘게 나눌 수 있다.
