# WZD 모바일 보드 가시성 버그 검증 보고서

## 1. 실제 문제
모바일에서 상단영역 수정 이후, 사용자가 보드 탭을 보더라도 실제 보드 내용이 화면에 나타나지 않는 회귀 버그가 발생했다.

사용자 관찰 기준:
- 보드 이름 칩은 보임
- 옆으로 슬라이드해도 보드 내용이 나오지 않음

즉 데이터 삭제가 아니라, **모바일에서 보드 콘텐츠 영역이 숨겨지거나 가려진 상태**일 가능성이 높다.

## 2. 왜 지금 중요한가
- 실제 사용 불가 수준의 회귀 버그다.
- 공모전/랜딩보다 우선순위가 높다.
- 사용자는 `내 보드가 사라졌다`고 느낄 수 있다.

## 3. 가장 의심되는 원인
- 모바일에서 상단 패널/컨테이너가 보드 영역을 밀어냄
- 보드 콘텐츠 컨테이너의 height / overflow / display 회귀
- 모바일 전용 CSS에서 본문이 숨겨지거나 높이가 0에 가깝게 계산됨
- 상단 구조를 단순화하는 과정에서 콘텐츠 영역 우선순위가 깨짐

## 4. 추천 MVP 범위
- 모바일에서 보드 콘텐츠가 다시 실제로 보이게 복구
- 상단은 유지하되 콘텐츠 영역을 가리지 않게 수정
- 레이아웃 회귀만 복구하고 기능 추가는 하지 않음

## 5. 지금 미뤄야 할 것
- 공모전용 추가 정리
- 상단 감성 개선
- 위젯/보드 UX 확장
- 랜딩 추가 개편

## 6. 성공 기준
1. 모바일에서 보드 탭을 선택하면 실제 보드 콘텐츠가 보인다.
2. 옆으로 스와이프/탭 전환 후에도 본문이 사라지지 않는다.
3. 상단 구조가 본문을 덮거나 숨기지 않는다.
4. build가 통과한다.
