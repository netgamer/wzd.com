# WZD 모바일 상단영역 검증 보고서

## 1. 실제 문제
현재 모바일 화면에서 상단영역은 다음 문제가 있다.
- 제목, 상태, 버튼, 탭, 입력 요소가 너무 가까이 모여 있어 첫인상이 답답하다.
- 저장 상태 메시지(특히 실패)가 지나치게 공격적으로 보이면 제품 신뢰를 깎는다.
- 사용자가 어디를 먼저 봐야 하는지 시선 우선순위가 약하다.

즉 문제는 기능 부족이 아니라 **상단영역 정보 밀도와 시각적 우선순위**다.

## 2. 왜 지금 중요한가
- 모바일 첫인상은 제품 신뢰에 직접 영향을 준다.
- 공모전이나 외부 평가에서도 모바일로 볼 가능성이 있다.
- 지금 상태를 방치하면 `기능은 많지만 정리 안 된 서비스`처럼 보일 수 있다.

## 3. 누구에게 먼저 중요한가
1. 모바일로 처음 보는 신규 사용자
2. 자주 들어와 빠르게 확인하려는 기존 사용자
3. 공모전 심사자 또는 외부 평가자

## 4. 리스크
- 상단을 너무 많이 줄이면 필요한 컨트롤까지 숨겨질 수 있다.
- 저장 상태를 너무 약하게 만들면 문제 인지가 어려워질 수 있다.
- 레이아웃 조정이 모바일 외 구간에 영향 줄 수 있다.

## 5. 추천 MVP 범위
- 모바일 상단영역의 텍스트/버튼 밀도 축소
- 저장 상태 메시지를 덜 공격적인 표현/위치로 조정
- 탭/액션/정보의 우선순위를 정리해 시선 흐름 개선

## 6. 지금 미뤄야 할 것
- 전체 보드 UX 재설계
- 기능 제거/이동 같은 큰 구조 변경
- 새 컴포넌트 체계 도입
- 모바일 외 전체 헤더 재설계

## 7. 성공 기준
1. 모바일 상단을 봤을 때 답답함이 줄어든다.
2. 저장 실패/상태 메시지가 덜 위협적으로 보이면서도 인지는 가능하다.
3. 사용자가 첫 시선에서 핵심 정보와 주 액션을 더 빨리 이해할 수 있다.

## 최종 판단
### PASS
지금 모바일 상단영역 정리는 우선순위가 높고, 작은 라운드로 바로 진행할 가치가 있다.
